# Runbook — `smb_loan_ai`

How to install, run, test, and debug the project. Copy-pasteable.

## Prerequisites

| Tool | Version | Install |
|---|---|---|
| **mise** | latest | `curl https://mise.run \| sh` (then add to shell rc) |
| **Python** | 3.12 (pinned in `mise.toml`) | `mise install python@3.12` (or auto-installed when you `cd` into the repo) |
| **uv** | latest | `mise install uv` |
| **Node.js** | 24 (pinned in `mise.toml`) | `mise install node@24` |
| **pnpm** | 10.33.3 (pinned in `package.json`) | `mise install pnpm` |
| **AWS credentials** | — | Configure `~/.aws/credentials` with a profile that has Bedrock access in `us-east-1`. |
| **`.env` file** | — | Create at repo root with `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASS`, `DB_NAME` for NeonDB. Local-only flow can run without these if you only use SQLite. |

`mise` will auto-resolve Python/Node/uv/pnpm versions from `mise.toml` once you `cd` into the repo.

## First-time setup

Run these in order from the repo root.

```bash
# 1. Activate mise (one-time, in your shell rc)
mise activate
mise trust

# 2. Create the Python virtual environment
mise run venv-create

# 3. Install all dependencies (Python via uv + Node via pnpm)
mise run inst

# 4. Download the local SQLite snapshot of the lending database
mise run download-db
#    → writes tmp/data.sqlite (~6.8 MB)

# 5. Create your .env (DB credentials for the cloud NeonDB)
#    Only needed if you want to query Postgres locally; SQLite works without it.
cp .env.example .env   # if .env.example exists; otherwise create manually
# Then fill in:
#   DB_HOST=ep-xxx-pooler.us-east-1.aws.neon.tech
#   DB_PORT=5432
#   DB_USER=neondb_owner
#   DB_PASS=...
#   DB_NAME=neondb

# 6. Verify AWS credentials
aws sts get-caller-identity --region us-east-1
#    → should return your account info
```

## Running locally

`mise run dev` is the canonical command. It first runs `mise run kill` (which calls `kill-dev-servers.py` to free ports 3000/3001/3002/8000/8080), then starts both servers concurrently.

**URLs once running:**

| Surface | URL |
|---|---|
| Next.js landing | `http://localhost:3000/` |
| Chat UI | `http://localhost:3000/chat` |
| FastAPI health | `http://localhost:8000/api/hello` |
| FastAPI chat endpoint | `POST http://localhost:8000/api/chat` (SSE) |
| Dev-only API test page | `http://localhost:3000/test-api` |

**First smoke test:** open `http://localhost:3000/chat` and type "What tables are in the database?" — the agent should call `get_database_schema` and return the schema text.

## Tests

- **Python**: `mise run test-python` → runs `pytest tests_python`.
  - Tests live in `tests_python/` mirroring the source tree. Note the doubled `one`: `tests_python/one/test_one_one_02_db.py` tests `smb_loan_ai/one/one_02_db.py`.
  - Coverage gaps: no integration test for `/api/chat`, no test for the full Strands `Agent.invoke()` flow, no test for the SQL tools end-to-end.
- **Node**: `mise run test-node` → runs `node --test tests_node/*.test.js`.
- **Coverage HTML**: `mise run cov-python` writes to `htmlcov/`. Do not commit this directory.

## Common failures and fixes

| Symptom | Likely cause | Fix |
|---|---|---|
| `mise run dev` says "port in use" | A previous dev session didn't shut down | `mise run kill`, then `mise run dev` again |
| Agent returns "credentials not found" locally | `~/.aws/credentials` missing or wrong profile | `aws configure` or set `AWS_PROFILE=...` before `mise run dev` |
| Agent runs but answers without calling tools | System prompt edited but server not restarted | The prompt is cached on first import — restart the FastAPI process |
| `tmp/data.sqlite` missing | Never ran `mise run download-db` | Run it. The file is gitignored. |
| Postgres connection fails | `.env` empty or wrong | Confirm `DB_*` vars; test with `psql` |
| `tool_write_debug_report` raises `AttributeError: path_debug_report_md` | `paths.py` missing that attribute | Add `path_debug_report_md = ...` to `PathEnum` |
| Frontend shows "Quota exceeded" | DynamoDB quota table hit its monthly cap | Either bump the cap in `quota.py` or wait until next month |
| Chat hangs / no streaming | Frontend version drift from `@ai-sdk/react` major | Inspect SSE in browser devtools; likely `ai_sdk_adapter.py` needs an update |

## Where to look when something breaks

- **FastAPI logs**: stdout of the `mise run fastapi-dev` process. Uvicorn `--reload` prints every reload + traceback on error.
- **Next.js logs**: stdout of `mise run next-dev`. React errors show up in browser devtools too.
- **Agent reasoning trace**: if `tool_write_debug_report` works, it dumps to `tmp/debug_report.md` after each invocation.
- **Bedrock invocation logs**: AWS CloudWatch (region `us-east-1`) — but you usually don't need them; check the local stdout first.
- **Quota state**: DynamoDB table (name set in `quota.py`).
- **DB queries**: SQLAlchemy doesn't log by default; flip `echo=True` on the engine constructor in `one_02_db.py` to dump every query.

## Vercel deployment notes

- `vercel.json` declares `api/**` as Serverless Functions (Python 3.12) and tells Vercel about pnpm corepack support.
- Env vars (DB and AWS credentials) are set in the Vercel project dashboard, not in the repo.
- Builds are triggered by `git push`. There's no separate "deploy" command in `mise.toml` — Vercel watches the Git remote.
- Cold starts hit the schema-reflection cache; expect the first request to a cold function to be ~2–4× slower.
